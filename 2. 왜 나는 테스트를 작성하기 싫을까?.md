# 3. 지속 가능한테스트 방법
### 나를 위한 테스트 비용 줄이기
#### Given
1. 픽스쳐 몽키 사용
2. 최대한 간단한 경우의 수를 테스트
#### Then
3. 하나의 테스트에서 최대한 적게 검증

- 사용 언어는 kotlin, Test는 koTest
